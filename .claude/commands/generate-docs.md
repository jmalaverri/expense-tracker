# Documentation Generator Command

You are a documentation generator. When invoked, you will produce two comprehensive documentation files for the feature described below.

## Input

**Feature Name**: $ARGUMENTS

## Instructions

### Step 1: Feature Analysis

Analyze the project to understand the feature:

1. Search the codebase for files related to the feature name: `find . -type f \( -name "*.ts" -o -name "*.tsx" -o -name "*.js" -o -name "*.jsx" -o -name "*.py" -o -name "*.go" -o -name "*.rs" -o -name "*.java" \) | xargs grep -li "$ARGUMENTS" 2>/dev/null`
2. Determine the **feature type** based on which files are found:
   - **Frontend**: Files in `src/components`, `src/pages`, `src/views`, `app/`, `pages/`, `frontend/`, or files with `.tsx`, `.jsx`, `.vue`, `.svelte` extensions
   - **Backend**: Files in `src/api`, `src/services`, `src/controllers`, `server/`, `backend/`, `routes/`, or files with `.go`, `.rs`, `.java`, `.py` (non-frontend) extensions
   - **Full-Stack**: Files found in BOTH frontend and backend directories
3. Identify the **tech stack** from `package.json`, `requirements.txt`, `go.mod`, `Cargo.toml`, or similar config files
4. Scan for existing documentation patterns in `docs/`, `README.md`, or any `*.md` files to match style conventions
5. Identify related features by scanning imports, shared modules, and cross-references

### Conditional Sections by Feature Type

- **Frontend-only**: Include sections on Component Props, State Management, Accessibility (ARIA), Responsive Breakpoints, and Browser Compatibility. Omit API Reference, Database Schema.
- **Backend-only**: Include sections on API Reference, Database Schema, Background Jobs, Queue/Event handling. Omit State Management, Component Props.
- **Full-Stack**: Include ALL sections.

### User Doc Conditional Sections by Feature Type

- **Frontend-only**: Focus on visual UI steps with screenshot placeholders for every interaction. Include keyboard shortcuts and accessibility notes.
- **Backend-only / API**: Reframe as an "Integration Guide" with code snippets in curl/Python/JS. Replace screenshot placeholders with request/response examples.
- **Full-Stack**: Provide the full UI walkthrough AND include an "API Access" appendix section with code examples.

### Step 4: Auto-Link Related Documentation

1. Scan all existing `.md` files in `docs/` for mentions of the feature name or related terms
2. Add cross-reference links in BOTH generated documents to any related docs found
3. If existing docs reference this feature, suggest (via stdout) that those docs should link back
